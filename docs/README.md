# Documentation System

Harnessing uses a lightweight engineering documentation system.

## Structure

- `docs/direction`: vision and roadmap
- `docs/planning`: backlog and execution priorities
- `docs/status`: current state and changelog
- `docs/testing`: repeatable verification steps
- `docs/tech-specs`: implementation and boundary specs
- `docs/tech-specs/TRANSCRIPT-INGESTION-SPEC.md`: transcript parsing and storage boundary
- `docs/tech-specs/PLATFORM-SUPPORT.md`: Windows-first support policy
- `docs/tech-specs/DUAL-TRACK-OPERATING-MODEL.md`: proving-ground and productization workflow
- `docs/tech-specs/CONSUMER-INTEGRATION-CONTRACT.md`: contract between Harnessing and consuming repos
- `docs/testing/TRANSCRIPT-INGESTION-TEST.md`: transcript ingestion verification steps
