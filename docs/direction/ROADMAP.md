# Roadmap

## Phase 1

- establish standalone repository structure
- package the CLI-first memory engine
- support doc ingestion, derived memory, search, bundle, and manual capture
- define the Windows-first support policy
- establish the dual-track proving-ground plus productization workflow

## Phase 2

- add transcript ingestion
- add tool-write event ingestion
- add stronger memory typing for decision, troubleshooting, technique, and reflection

## Phase 3

- add HTTP API and machine-readable export formats
- support direct integration with Codex hooks and Claude wrappers
- support multi-project workspace memory
